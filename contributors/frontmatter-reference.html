Frontmatter Reference

Every frontmatter field, what it does, and when to use it.

560 words·3 min read

Every .mdx file in content/docs/ must have a frontmatter block. Fields are validated at build time — an invalid or missing required field fails the build with a clear error message pointing at the file.

Required fields

title

Shown as the page <h1>, in the browser tab, and in search results. Do not repeat it as a heading inside the page body.

Optional fields

description

One sentence. Shown in gray under the title and used as the meta description for search engines.

type

The Diátaxis content type. Drives rendering hints and helps readers know what to expect before they start reading.

ValueWhen to use
tutorialTeaching something — reader is learning
how-toSolving a specific problem — reader has a goal
referenceFactual lookup — reader knows what they want
explanationBackground understanding — reader wants context
quickstartFastest path to a working result
changelogRelease notes
faqQuestion-and-answer format
glossaryTerm definitions
cookbookCopy-pasteable recipes, minimal prose
architectureHow the system itself is built (contributor-facing)
migrationVersion upgrade instructions

section

Which top-level tab this page belongs to. Must match a section id in pepa.config.ts.

ValueTab
docsMain docs tab (default)
api-referenceAPI Reference tab
cookbookCookbook tab
contributorsContributors tab

nav

The sidebar group label within the section. Valid values:

Tutorials · How-to Guides · Reference · Explanation · Quickstart · Changelog · FAQ · Glossary

order

Integer. Position within the sidebar group — lower numbers sort first. Pages with the same order sort alphabetically by title.

status

Adds a colored badge next to the page title. Omit this field for stable pages — stable is the default and gets no badge.

ValueBadge colorWhen to use
betaBlueFeature works but the API or content may change
experimentalOrangeMay change significantly or be removed
deprecatedRedWill be removed; link to the replacement

navParent

Creates a collapsible sub-section within the sidebar group. All pages that share the same nav and navParent values are grouped under a twirl-open heading. The sub-section opens automatically when the reader is on one of its pages.

Omit this field to have the page appear directly in the top level of its nav group with no nesting.

hidden

Excludes the page from the sidebar and nav tree. The page still exists at its URL and is included in the build — useful for landing pages or pages linked from elsewhere but not meant to appear in navigation.

copyable

Shows a Copy page button in the top-right corner of the content header. Clicking it copies the full rendered text of the page to the clipboard — useful for readers who want to paste the content into an AI assistant, a notes app, or another document.

The button is off by default. Add copyable: true to any page where sharing the full content in one click would be genuinely useful — long reference pages, tutorials, and cookbook recipes are good candidates. Avoid it on pages that are mostly navigation or placeholder content.

💡

The copy captures the rendered text (what readers see), not the raw MDX source. Code blocks, table content, and component output are all included.
