Style Linting

How to run the Google Developer Style Guide linter and interpret its output.

203 words·2 min read

Pepa uses Vale with the Google Developer Style Guide ruleset to catch common style violations before they reach production.

First-time setup

vale:sync downloads rules into .vale/styles/ at the repo root. That directory is gitignored — run vale:sync once after cloning.

Reading the output

apps/docs/content/docs/getting-started.mdx
 14:12  warning  Use 'select' instead of 'click on'.  Google.We
 27:4   warning  Avoid using 'simply'.                Google.Simplicity

Each line shows: line:col severity message rule-name.

Severity levels (configured in .vale.ini):

LevelMeaning
suggestionWorth considering; low urgency
warningLikely a style issue; fix before merging
errorMust fix

The current config (MinAlertLevel = suggestion) shows all levels.

Common violations

Vale saysFix
"click on"Use "click"
"simply", "easy", "just"Remove — they're dismissive
Passive voice detectedRewrite to active voice
Heading not in sentence caseCapitalize only the first word and proper nouns
Oxford comma missingAdd the comma before "and" in a list
Future tense ("will")Use present tense where possible
