## Steps

Numbered steps with a connecting line. Code blocks, callouts, and any other
content can live inside a step.

<Steps>
  <Step title="Install dependencies">
    ```bash
    pnpm install
    ```
  </Step>
  <Step title="Configure your project">
    Edit `apps/docs/pepa.config.ts` to set your project name and enable the
    sections you need.
  </Step>
  <Step title="Add your first doc">
    Create an `.mdx` file under `content/docs/` with the required frontmatter,
    then run `pnpm dev` to see it appear in the sidebar.

    ```yaml
    ---
    title: My First Page
    nav: Tutorials
    order: 1
    ---
    ```

  </Step>
  <Step title="Deploy">
    Push to `main`. GitHub Actions builds the static export and publishes it
    to GitHub Pages automatically.
  </Step>
</Steps>

## Accordion

Collapsible sections. Uses native `<details>` — no JavaScript required.

<Accordion>
  <AccordionItem title="What is Diátaxis?">
    Diátaxis is a framework that organises documentation into four types:
    **Tutorials**, **How-to Guides**, **Reference**, and **Explanation**.
    Each type serves a different user need.
  </AccordionItem>
  <AccordionItem title="Do I need to use all four types?" open>
    No. All four Diátaxis sections (and the special types like Cookbook) are
    **opt-in**. Set `enabled: false` in `pepa.config.ts` to hide any section
    you don't need.
  </AccordionItem>
  <AccordionItem title="Can I nest pages in the sidebar?">
    Yes — add a `navParent` field to your frontmatter to group related pages
    under a collapsible heading within a nav group:

    ```yaml
    nav: Reference
    navParent: Authentication
    ```

  </AccordionItem>
</Accordion>

## Nested nav sidebar

Pages can be grouped under a collapsible sub-section within any nav group.
Add `navParent` to the page's frontmatter:

```yaml
---
title: Token Types
nav: Reference
navParent: Authentication
order: 1
---
```

The sidebar renders the sub-section collapsed by default. It auto-opens when
the active page lives inside it.
