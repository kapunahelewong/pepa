# Getting Started

This is a sample page to prove the content pipeline works end to end:
frontmatter validation, MDX compilation, syntax highlighting, and a custom
component imported from the shared `docs-ui` package.

<Callout type="tip">
  Edit this file at `content/docs/getting-started.mdx` — saving it hot-reloads
  in dev and re-validates against the schema in `content-collections.ts`.
</Callout>

## Install

```bash
pnpm install
pnpm dev
```

## Why the schema matters

If you delete the `title` field above and save, the build fails immediately
with a clear error pointing at this file — instead of shipping a docs page
with a blank `<h1>` to production. That's the whole value of a typed content
layer over parsing MDX by hand.
