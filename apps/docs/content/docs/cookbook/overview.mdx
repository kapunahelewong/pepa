Recipes are short, complete, and runnable. Each one solves exactly one problem.
For background on *why* something works the way it does, see the main docs.

<Callout type="tip">
  Every recipe uses the [JSONPlaceholder](https://jsonplaceholder.typicode.com)
  API as the example. Swap in your own endpoints and the patterns transfer directly.
</Callout>

| Recipe | What it covers |
|--------|---------------|
| [Authenticate a request](/cookbook/authenticate-a-request) | Add a bearer token to every request |
| [Paginate through results](/cookbook/paginate-results) | Walk a cursor-paginated list endpoint |

## How recipes are structured

Each recipe has:

1. **The goal** — one sentence
2. **The code** — complete, copy-pasteable
3. **What to change** — the two or three lines you'll need to adapt

Nothing else. If you want context, the main docs have it.
